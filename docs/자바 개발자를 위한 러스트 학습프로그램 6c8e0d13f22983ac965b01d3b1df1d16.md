## Java와 Rust 개념 매핑✔️

| **자바 (Java)** | **러스트 (Rust)** | **특징** |
| --- | --- | --- |
| **Class** | **Struct + impl 블록** | Rust는 데이터와 행동을 분리하여 정의함 |
| **Field** (멤버 변수) | **Field** (구조체 필드) | 데이터를 저장하는 공간 |
| **Method** (인스턴스 메서드) | **Method** (첫 인자가 `self`인 함수) | 인스턴스를 통해 호출하는 함수 (`user.login()`) |
| **Static Method** | **Associated Function** (`self`가 없는 함수) | 구조체 이름으로 직접 호출하는 함수 (`User::new()`) |
| **this** 키워드 | **self** 매개변수 | 자기 자신 인스턴스를 가리키는 포인터/참조 |

## Rust에서의 3대 프로그래밍 패러다임 비교✔️

Clean Code에 언급된 세 가지 패러다임이 Rust에서 어떻게 구현되고 어떤 평가를 받는지 비교해 드리겠습니다.

### ① 절차지향 스타일 (Procedural)

- **개념:** 위에서 아래로 순차적으로 명령어를 실행하고, 상태를 전역/지역 변수로 제어하는 방식입니다.
- **Rust에서의 모습:** 수많은 `mut` 변수, `for`/`while` 루프, 명령형 조건문(`if`)을 남발하는 코드입니다.
- **선호도 (낮음):** Rust에서도 당연히 작동하지만, 가변 참조자(`&mut`)를 너무 많이 만들면 Rust의 대여 검사기(Borrow Checker)와 싸우게 되어 코딩이 매우 고통스러워집니다. 시스템 프로그래밍의 극단적인 최적화 영역이 아니라면 메인 로직에서는 지양됩니다.

### ② 객체지향 스타일 (Object-Oriented)

- **개념:** 데이터와 소스 코드를 캡슐화하여 객체 간의 상호작용으로 시스템을 구성하는 방식입니다.
- **Rust에서의 모습:** `struct`로 필드를 정의하고 `impl`로 메서드를 붙이며, `trait`로 다형성을 구현합니다.
- **선호도 (보통~높음):** 단, **자바식 OOP(클래스 상속)는 불가능**합니다. Rust에는 `extends`가 없습니다. 구조체는 오직 다른 트레이트를 인터페이스로서 구현(`implements`)할 수만 있습니다. 따라서 상속보다는 **조합(Composition)과 트레이트 기반 다형성**을 적극적으로 활용하는 OOP 스타일을 따릅니다.

### ③ 함수형 스타일 (Functional)

- **개념:** 불변성(Immutability)을 지키고, 함수를 일급 시민으로 취급하며, 부수 효과(Side Effect)를 최소화하는 파이프라인 방식입니다.
- **Rust에서의 모습:** 불변 변수(`let`), 패턴 매칭(`match`, `if let`), 고차 함수와 이터레이터 패턴, 클로저(`|x| x + 1`)를 활용합니다.
- **선호도 (매우 높음):** Rust 개발자들이 가장 사랑하는 스타일입니다. Rust의 모든 제어문(`if`, `match`, 블록 `{}`)은 값을 반환하는 표현식(Expression)입니다. 이 특성과 `Option`/`Result`가 결합하여 에러 처리와 데이터 가공을 함수형 파이프라인으로 안전하고 간결하게 끝내는 것을 최고로 여깁니다.

### 한눈에 보는 패러다임 비교 테이블

| **비교 항목** | **절차지향 스타일** | **객체지향 스타일 (Rust 방식)** | **함수형 스타일** |
| --- | --- | --- | --- |
| **핵심 키워드** | `mut`, `for / while`, 제어 흐름 | `struct`, `impl`, `trait`, 조합 | `Iterator`, `Closure`, `match`, 표현식 |
| **상태 관리** | 가변 변수 직접 변경 | 구조체 필드 캡슐화 | 불변 데이터, 새로운 상태 반환 |
| **다형성 구현** | 없음 (함수 분기) | **트레이트 객체 (`&dyn Trait`)** | 제네릭 정적 디스패치 (`<T>`) |
| **자바와의 매칭** | C 스타일의 고전적 자바 코드 | 인터페이스와 포장(Wrapper) 중심 | Java 8+ Stream, Optional, Lambda |
| **Rust 실무 평가** | 필요할 때만 제한적 사용 | **전체적인 프로그램 설계 아키텍처** | **상태 변경 및 데이터 가공 로직의 표준** |

## 표준 라이브러리 std✔️

Rust의 표준 라이브러리(`std`)는 운영체제와 직접 소환하여 고성능 시스템 프로그램을 안전하게 작성할 수 있도록 돕는 핵심 도구 모음입니다. 

## 요약 목록

- **`std::prelude` (자동 임포트):** 기본 구조체와 필수 트레이트(`Option`, `Result`, `Clone` 등)가 별도 선언 없이 제공됩니다.
- **코어 컬렉션 및 문자열:** 힙 메모리를 활용하는 동적 배열(`Vec`), 키-값 맵(`HashMap`), 그리고 문자열 구조(`String`)가 포함됩니다.
- **I/O 및 시스템 통신:** 파일 시스템 접근, 버퍼 기반 입출력, 크로스 플랫폼 동시성(`Thread`, `Mutex`) 제어를 지원합니다.

## 1. 최상위 핵심 모듈 구성

### 1) 프리루드 (Prelude)

Rust 코드를 작성할 때 `use` 문으로 명시하지 않아도 항상 자동으로 로드되는 에센셜 컴포넌트들입니다.

- `Option<T>` / `Result<T, E>`
- 기본 변환 트레이트: `Into`, `From`, `AsRef`
- 메모리 및 라이프사이클 관리: `Clone`, `Drop`

### 2) 메모리 관리 및 동적 주소 제어 (`std::boxed`, `std::rc`, `std::cell`)

가비지 컬렉터(GC)가 없는 Rust에서 메모리 소유권을 유연하게 제어하기 위한 포인터 라이브러리입니다.

- **`Box<T>`:** 힙(Heap) 메모리에 데이터를 할당하는 기본 포인터입니다. (Java의 일반 참조형 객체 생성과 유사)
- **`Rc<T>` / `Arc<T>`:** 참조 카운팅을 통해 하나의 데이터를 여러 곳에서 안전하게 공유(Read-only)할 수 있게 합니다. `Arc`는 멀티스레드 환경용(Thread-safe)입니다.
- **`Cell<T>` / `RefCell<T>`:** 컴파일러의 대여 검사를 우회하여, 불변 참조 내부의 데이터를 런타임에 수정할 수 있는 '내부 가변성'을 제공합니다.

## 2. 데이터 가공 및 수집 (`std::collections`, `std::iter`)

자바의 컬렉션 프레임워크와 `java.util.stream`에 직접 대응하는 도구들입니다.

```
┌─────────────────────────┬─────────────────────────┐
│     Java Collection     │     Rust Counterpart    │
├─────────────────────────┼─────────────────────────┤
│ ArrayList<T>            │ Vec<T>                  │
│ HashMap<K, V>           │ HashMap<K, V>           │
│ HashSet<T>              │ HashSet<T>              │
│ Stream<T>               │ Iterator                │
└─────────────────────────┴─────────────────────────┘
```

- **`Vec<T>`:** 연속된 메모리 공간을 사용하는 동적 배열입니다. 실무에서 데이터 배열을 다룰 때 90% 이상 사용됩니다.
- **`HashMap<K, V>`:** 기본적으로 보안에 강한 암호화 해시 알고리즘(SipHash)을 사용하여 데이터를 저장하는 키-값 맵입니다.
- **`Iterator` 트레이트:** 루프 앤 필터 파이프라인을 구축하는 핵심 명세입니다. `.map()`, `.filter()`, `.fold()` 등 자바의 스트림과 닮은 고차 함수를 제공하며, 컴파일 시 내부 루프로 완벽히 최적화되어 속도가 매우 빠릅니다.

## 3. 문자열 처리 (`std::string`, `std::str`)

Rust 문자열은 메모리 소유 관계에 따라 크게 두 가지로 분리되어 설계되었습니다.

- **`String`:** 힙 영역에 할당되어 크기를 동적으로 늘리거나 줄일 수 있는 소유권 체계의 문자열 객체입니다.
- **`str` (보통 `&str` 형태로 사용):** 문자열의 어딘가를 가리키는 고정 크기의 읽기 전용 슬라이스 참조자입니다. 메모리 복사 없이 문자열 데이터의 일부분을 가볍게 참조할 때 사용합니다.

## 4. 입출력 및 동시성 제어 (`std::io`, `std::fs`, `std::thread`)

- **`std::io` (Read, Write, BufReader):** 바이트 스트림 입력과 출력을 추상화한 트레이트 집합입니다. 버퍼링을 제공하는 `BufReader`/`BufWriter`를 결합하여 효율적인 하드웨어 IO를 수행합니다.
- **`std::fs`:** 파일 생성, 읽기, 쓰기 및 디렉터리 제어 등 운영체제 파일 시스템과의 상호작용을 담당합니다.
- **`std::thread` & `std::sync`:** OS 원시 스레드를 생성하는 `thread::spawn`과 스레드 간 데이터 격리 및 가변 공유를 지원하는 `Mutex`, `RwLock`, `mpsc` (메시지 패싱 채널) 구조를 지원합니다.
