# 변수 선언(let)과 불변성(Immutability), 가변성(mut)

- **요약 목록**
    - **기본 불변성**: Rust의 변수는 기본적으로 변경 불가능(`immutable`)하며, 값을 바꾸려면 컴파일 에러가 발생합니다.
    - **가변성 지정**: 값을 변경해야 하는 변수는 `mut` 키워드를 명시적으로 붙여서 선언해야 합니다.
    - **안전성 확보**: 기본을 불변으로 지정함으로써 동시성 환경에서 데이터 경합을 방지하고 버그를 예방합니다.

## 변수 선언과 불변성 (Immutability)

Rust는 안전성을 최우선으로 설계된 언어입니다. 그렇기 때문에 `let` 키워드로 선언한 변수는 **기본적으로 값을 변경할 수 없는 불변(Immutable) 상태**가 됩니다.

```rust
let x = 5;
x = 6; // 컴파일 에러 발생!
```

이미 할당된 불변 변수에 새로운 값을 할당하려고 하면, Rust 컴파일러는 상냥하지만 단호하게 "재할당할 수 없다"는 에러를 던지며 빌드를 거부합니다. 코드의 다른 부분에서 값이 예측 불가능하게 바뀌는 것을 원천 차단하기 위함입니다.

## 가변성 (Mutability) 선언

변수의 값을 변경해야 하는 상황이 오면 `mut` 키워드를 `let` 뒤에 추가하여 컴파일러에게 이 변수는 수정될 수 있음을 명시적으로 알려야 합니다.

```rust
let mut x = 5;
x = 6; // 정상 작동
```

`mut`을 사용하면 이 코드를 읽는 다른 개발자나 컴파일러 모두에게 "이 변수는 앞으로 값이 바뀔 가능성이 있다"는 힌트를 제공하게 되므로, 코드의 의도를 파악하기가 훨씬 수월해집니다.
