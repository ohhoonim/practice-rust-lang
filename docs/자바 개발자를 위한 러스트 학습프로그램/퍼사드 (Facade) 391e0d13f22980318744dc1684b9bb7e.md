# 퍼사드 (Facade)

- **목표:** 여러 복잡한 서브시스템 구조체들의 상호작용을 숨기고, 클라이언트가 단 하나의 단순화된 메서드 호출만으로 원하는 기능을 수행할 수 있도록 하는 퍼사드 패턴을 구현합니다.
- **조건:** 퍼사드 구조체는 복잡한 서브시스템 객체들을 내부에 소유하거나 참조하며, 복잡한 초기화 및 호출 순서를 하나의 고수준 인터페이스(메서드)로 캡슐화해야 합니다.

### [연습 문제] 퍼사드 패턴 구현

컴퓨터를 켤 때 일어나는 복잡한 과정(CPU 초기화, 메모리 로드, 하드디스크 읽기 등)을 클라이언트가 버튼 하나로 실행할 수 있도록 `ComputerFacade`를 구현하세요.

1. 서브시스템: `Cpu`, `Memory`, `HardDrive` 구조체는 각각 복잡한 하위 제어 메서드를 가집니다. (이 코드는 건드리지 않습니다.)
2. 퍼사드: `ComputerFacade` 구조체는 세 서브시스템을 필드로 가지며, `start_computer`라는 하나의 고수준 메서드를 제공합니다.
3. `start_computer` 메서드는 CPU 부팅, 메모리 로딩, 하드디스크 실행 순서를 올바르게 조합하여 하나의 통합된 메시지(String)를 반환합니다.

```rust
// 1. 복잡한 서브시스템들 (Subsystems)
struct Cpu;
impl Cpu {
    fn freeze(&self) -> String { "CPU frozen.".to_string() }
    fn jump(&self) -> String { "CPU jumped to boot address.".to_string() }
    fn execute(&self) -> String { "CPU executing commands.".to_string() }
}

struct Memory ;
impl Memory {
    fn load(&self) -> String { "Memory loaded bootloader.".to_string()}
}

struct HardDrive;
impl HardDrive {
    fn read_boot_sector(&self) -> String { "HardDrive read boot sector.".to_string()}
}

// 2. 고수준 인터페이스를 제공하는 퍼사드 객체 (Facade)
struct ComputerFacade {
    cpu: Cpu,
    memory: Memory,
    hard_drive: HardDrive,
}

impl ComputerFacade {
    fn new() -> Self {
        ComputerFacade {
            cpu: Cpu,
            memory: Memory,
            hard_drive: HardDrive,
        }
    }

    // start_computer 메서드를 구현하세요.
    // 내부 서브시스템들을 다음 순서로 호출하고 각 결과를 줄바꿈('\n')으로 연결하여 하나의 String으로 반환합니다.
    // 순서:
    // 1. cpu.freeze()
    // 2. memory.load()
    // 3. hard_drive.read_boot_sector()
    // 4. cpu.jump()
    // 5. cpu.execute()
    fn start_computer(&self) -> String {
        // 이 부분을 구현하세요.
    }
}

#[cfg(test)]
mod tests {
    use super::*;

    #[test]
    fn test_facade_pattern() {
        let computer = ComputerFacade::new();
        let boot_log = computer.start_computer();

        let expected_log = concat!(
            "CPU frozen.\n",
            "Memory loaded bootloader.\n",
            "HardDrive read boot sector.\n",
            "CPU jumped to boot address.\n",
            "CPU executing commands."
        );

        assert_eq!(boot_log, expected_log);
    }
}
```
