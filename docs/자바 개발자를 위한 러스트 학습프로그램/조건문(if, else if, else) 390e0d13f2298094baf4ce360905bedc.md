# 조건문(if, else if, else)

- **요약 목록**
    - **조건의 타입**: `if` 문의 조건식은 반드시 부울형(`bool`)이어야 합니다. 타 언어처럼 정수나 객체를 조건식에 그대로 사용할 수 없습니다.
    - **식(Expression)으로서의 if**: `if` 문은 값을 반환하는 표현식으로 사용할 수 있어, 변수에 대입할 수 있습니다.
    - **타입 일치**: `if` 표현식을 변수에 대입할 때, 각 가지(`arm`)가 반환하는 값의 타입은 모두 일치해야 합니다.

## 기본적인 조건문 구조

Rust의 `if` 문은 기본적으로 다른 언어와 유사하지만, **조건식을 괄호(`()`)로 감싸지 않는다**는 특징이 있습니다. 단, 조건에 따른 실행 블록 `{}`은 필수입니다.

```rust
let number = 3;

if number < 5 {
    println!("조건이 참입니다.");
} else if number == 5 {
    println!("숫자가 5입니다.");
} else {
    println!("조건이 거짓입니다.");
}
```

### 주의: 엄격한 부울형(`bool`) 검사

Rust는 정수형 등을 자동으로 부울형으로 변환하지 않습니다. 조건식은 반드시 `bool` 값이어야 합니다.

```rust
let number = 3;

// 컴파일 에러! (기존 타 언어처럼 0이 아니라고 참으로 인정하지 않음)
if number {
    println!("숫자가 존재합니다.");
}

// 올바른 방법
if number != 0 {
    println!("숫자가 0이 아닙니다.");
}
```

## `let` 구문에서 `if` 사용하기

**`if`는 구문(Statement)이 아니라 값을 반환하는 표현식(Expression)입니다**. 따라서 `let` 변수를 선언할 때 우변에 `if` 문을 바로 사용할 수 있습니다.

```rust
let condition = true;
let number = if condition { 5 } else { 6 };
```

### 변수 대입 시 주의사항

`if` 표현식을 변수에 바인딩할 때, `if` 블록과 `else` 블록이 반환하는 **데이터 타입은 반드시 일치**해야 합니다. Rust는 컴파일 시점에 변수의 타입을 명확히 알아야 하기 때문입니다.

```rust
// 컴파일 에러! (i32와 &str로 반환 타입이 서로 다름)
let number = if condition { 5 } else { "six" };
```
