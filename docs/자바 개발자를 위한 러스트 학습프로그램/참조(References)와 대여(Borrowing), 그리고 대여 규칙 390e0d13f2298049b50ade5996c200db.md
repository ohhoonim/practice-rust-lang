# 참조(References)와 대여(Borrowing), 그리고 대여 규칙

- **요약 목록**
    - **참조(`References`)**: 소유권을 가져오지 않고, 주소값만 가리켜 데이터에 접근할 수 있게 하는 기법입니다. 앰퍼샌드(`&`) 부호를 사용합니다.
    - **대여(`Borrowing`)**: 함수 매개변수 등으로 참조를 전달하는 행위를 '값을 대여했다'고 표현합니다.
    - **대여 규칙**:
        1. 단 하나만의 가변 참조(`&mut`)를 가질 수 있거나, 혹은 여러 개의 불변 참조(`&`)를 가질 수 있습니다 (동시 존재 불가).
        2. 참조는 항상 유효해야 합니다 (대상을 잃은 참조 금지).

## 참조(References)와 대여(Borrowing)

앞서 배운 '이동(Move)'으로 인해, 함수에 변수를 넘겨줄 때마다 소유권이 넘어가 버려 매번 다시 반환받아야 하는 번거로움이 생깁니다. 이를 해결하기 위해 Rust는 소유권을 넘기지 않고 값에 접근할 수 있는 참조(References)를 제공합니다.

```rust
fn calculate_length(s: &String) -> usize { // &String 타입을 받음
    s.len()
} // s는 참조일 뿐이므로 스코프가 끝나도 실제 문자열 메모리가 해제(drop)되지 않음
```

- `&` 기호가 바로 참조를 나타냅니다. 소유권을 가져오지 않고 단지 값을 가리키기만 합니다.
- 이처럼 참조를 만드는 행위를 Rust에서는 대여(Borrowing)라고 부릅니다. 현실에서 물건을 빌리면(대여) 쓰고 나서 돌려주어야 하듯, 스코프가 끝나면 참조는 자동으로 반환됩니다.

## 가변 참조 (Mutable References)

기본적인 참조(`&`)는 값을 읽기만 할 수 있을 뿐, 수정할 수 없습니다. 대여한 값을 수정하고 싶다면 가변 참조(`&mut`)를 사용해야 합니다.

```rust
let mut s = String::from("hello");
change(&mut s); // 가변 참조 전달

fn change(some_string: &mut String) {
    some_string.push_str(", world");
}
```

## 대여 규칙 (The Rules of Borrowing)

Rust가 컴파일 시점에 메모리 안전성을 완벽하게 보장하기 위해 강제하는 매우 중요한 대여 규칙 두 가지가 있습니다.

### 규칙 1. 데이터 경합 방지 규칙

어떤 데이터에 대해 특정 스코프 내에서 다음 두 가지 조건 중 **하나만** 만족해야 합니다.

- **어떠한 개수든 상관없이 불변 참조(`&T`)들을 가질 수 있습니다.**
- **오직 단 하나만의 가변 참조(`&mut T`)를 가질 수 있습니다.**

즉, 누군가 값을 읽고 있는 동안(`&`)에는 아무도 그 값을 바꿀 수 없고, 누군가 값을 바꾸고 있는 동안(`&mut`)에는 아무도 그 값을 읽거나 바꿀 수 없습니다.

```rust
let mut s = String::from("hello");

let r1 = &s; // 불변 참조 1 (OK)
let r2 = &s; // 불변 참조 2 (OK)
// let r3 = &mut s; // 컴파일 에러! 이미 불변 참조로 대여 중이므로 가변 대여 불가능
```

### 규칙 2. 유효하지 않은 참조(Dangling Reference) 방지

참조는 자신이 가리키고 있는 실제 값보다 오래 살아남을 수 없습니다.

```rust
// 컴파일 에러가 발생하는 코드 예시
fn dangle() -> &String {
    let s = String::from("hello");
    &s // s는 함수가 끝나면 메모리에서 사라지는데, 사라질 메모리의 주소를 반환하려고 함
}
```

Rust 컴파일러는 가리키는 대상이 먼저 사라져 허공을 가리키는 위험한 참조가 발생하는 것을 방지합니다.
