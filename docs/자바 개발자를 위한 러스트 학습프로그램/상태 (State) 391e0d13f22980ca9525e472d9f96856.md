# 상태 (State)

- **목표:** 객체의 내부 상태를 캡슐화된 상태 객체들로 분리하고, 상태 변화에 따라 동일한 메서드 호출이 완전히 다른 행동을 하도록 만드는 상태 패턴을 구현합니다.
- **조건:** Rust에서는 상태의 전환을 표현하기 위해 `enum`과 매치 표현식(Pattern Matching)을 사용하거나, 상태를 트레이트 객체(`Box<dyn State>`)로 동적 위임하는 방식을 사용할 수 있습니다. 여기서는 문맥(Context) 내부에서 구체적인 상태를 트레이트 객체로 전환하는 정석적인 구조를 연습합니다.

### [연습 문제] 상태 패턴 구현

소형 가전기기인 선풍기(Fan)의 풍량 조절 시스템을 상태 패턴으로 구현하세요.

1. **State 인터페이스:** `State` 트레이트는 버튼을 누를 때의 동작을 정의하는 `press_button` 메서드를 가집니다.
2. **구체적인 상태:** * `OffState`: 버튼을 누르면 "Low Speed"로 변경되며 다음 상태로 `LowState`를 반환합니다.
    - `LowState`: 버튼을 누르면 "High Speed"로 변경되며 다음 상태로 `OffState`를 반환합니다.
3. **Context 객체:** `Fan` 구조체는 내부에 `Option<Box<dyn State>>` 필드를 가지며, 버튼을 누를 때마다 현재 상태의 `press_button`을 호출하고 반환된 새 상태로 갱신합니다.

```rust
// 1. 상태 인터페이스 (State)
trait State {
    fn press_button(&self) -> (String, Box<dyn State>);
}

// 2. 구체적인 상태 - Off (Concrete State)
struct OffState;
impl State for OffState {
    // press_button을 구현하세요.
    // 반환값의 첫 번째 요소(로그)는 "Turning fan on to low speed."로 지정하고,
    // 두 번째 요소(다음 상태)는 Box::new(LowState)를 반환합니다.
    fn press_button(&self) -> (String, Box<dyn State>) {
        // 이 부분을 구현하세요.
    }
}

// 3. 구체적인 상태 - Low (Concrete State)
struct LowState;
impl State for LowState {
    // press_button을 구현하세요.
    // 반환값의 첫 번째 요소(로그)는 "Turning fan off."로 지정하고,
    // 두 번째 요소(다음 상태)는 Box::new(OffState)를 반환합니다.
    fn press_button(&self) -> (String, Box<dyn State>) {
        // 이 부분을 구현하세요.
    }
}

// 4. 문맥 객체 (Context)
struct Fan {
    state: Option<Box<dyn State>>,
}

impl Fan {
    fn new() -> Self {
        Fan {
            state: Some(Box::new(OffState)),
        }
    }

    // press_button 메서드를 구현하세요.
    // 1. self.state에서 take()를 이용해 상태를 꺼내옵니다.
    // 2. 꺼내온 상태의 press_button()을 호출합니다.
    // 3. 반환된 새 상태를 다시 self.state에 주입(Some)합니다.
    // 4. 반환된 로그 문자열(String)을 최종 반환합니다.
    fn press_button(&mut self) -> String {
        // 이 부분을 구현하세요.
    }
}

#[cfg(test)]
mod tests {
    use super::*;

    #[test]
    fn test_state_pattern() {
        let mut fan = Fan::new();

        // 초기 상태: Off -> 버튼 클릭 -> Low 상태로 전환
        assert_eq!(fan.press_button(), "Turning fan on to low speed.");

        // 현재 상태: Low -> 버튼 클릭 -> Off 상태로 전환
        assert_eq!(fan.press_button(), "Turning fan off.");

        // 다시 Off -> 버튼 클릭 -> Low 상태로 전환
        assert_eq!(fan.press_button(), "Turning fan on to low speed.");
    }
}
```
