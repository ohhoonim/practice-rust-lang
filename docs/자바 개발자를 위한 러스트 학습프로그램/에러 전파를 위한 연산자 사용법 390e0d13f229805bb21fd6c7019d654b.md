# 에러 전파를 위한 ? 연산자 사용법

- **요약 목록**
    - **`?` 연산자의 역할**: 에러가 발생했을 때 이를 직접 처리하지 않고, 함수를 호출한 상위 문맥으로 에러를 위임(전파)합니다.
    - **동작 방식**: `Result`가 `Ok`이면 내부 값을 반환하고, `Err`이면 `return Err(...)`을 실행하는 것과 동일하게 작동하여 코드를 획기적으로 줄여줍니다.
    - **사용 조건**: `?` 연산자는 매칭되는 반환 타입(예: `Result` 또는 `Option`)을 가진 함수 내부에서만 사용할 수 있습니다.

## 에러 전파 (Propagating Errors)란?

함수를 구현할 때 에러가 발생할 수 있는 작업을 수행하는 경우, 이 에러를 함수 내부에서 직접 수습(`match`나 `panic!`)하지 않고, **이 함수를 호출한 쪽(상위 코드)에서 에러를 결정하도록 넘겨버리는 것**을 에러 전파라고 합니다.

## `?` 연산자를 이용한 간결한 에러 전파

기존의 `match` 문을 사용해 에러를 상위로 전파하려면 코드가 매우 길어집니다.

```rust
// match를 사용한 긴 에러 전파 예시
fn read_username_from_file() -> Result<String, io::Error> {
    let username_file_result = File::open("hello.txt");

    let mut username_file = match username_file_result {
        Ok(file) => file,
        Err(e) => return Err(e), // 에러를 감싸서 조기 리턴(전파)
    };
    // ... 후속 작업
}
```

Rust는 이 패턴을 획기적으로 줄여주는 **`?` 연산자**를 제공합니다. `Result` 값 뒤에 `?`를 붙이면 위의 `match` 문과 정확히 동일한 작동을 숏컷으로 수행합니다.

```rust
// ? 연산자를 사용한 간결한 에러 전파
fn read_username_from_file() -> Result<String, io::Error> {
    let mut username_file = File::open("hello.txt")?; // 실패 시 즉시 에러 반환됨
    let mut username = String::new();
    username_file.read_to_string(&mut username)?;     // 두 번째 전파
    Ok(username)
}
```

### `?` 연산자의 내부 동작

1. `File::open("hello.txt")` 결과가 `Ok(file)`이면, `?` 연산자는 `Ok` 내부의 실제 파일 객체를 `username_file` 변수에 쏙 빼내어 할당합니다.
2. 결과가 `Err(e)`이면, 함수 전체를 **그 즉시 조기 종료(return)** 시키고 호출한 상위 함수로 `Err(e)`를 던집니다.

## 함수 연결하기 (Chaining)

`?` 연산자 덕분에 에러가 발생할 수 있는 여러 메서드 호출을 한 줄로 우아하게 연결할 수 있습니다.

```rust
fn read_username_from_file_short() -> Result<String, io::Error> {
    let mut username = String::new();
    File::open("hello.txt")?.read_to_string(&mut username)?;
    Ok(username)
}
```

## `?` 연산자를 사용할 수 있는 곳

`?` 연산자는 **아무 데서나 쓸 수 없습니다.** `?`가 에러를 만나 매크로처럼 `return Err(...)`을 수행해야 하므로, **`?`를 사용하는 함수의 반환 타입이 `Result` 또는 `Option`과 같이 호환되는 타입**이어야만 컴파일이 가능합니다. 반환 타입이 `()`인 일반 함수 내부에서 쓰면 컴파일 에러가 발생합니다.
