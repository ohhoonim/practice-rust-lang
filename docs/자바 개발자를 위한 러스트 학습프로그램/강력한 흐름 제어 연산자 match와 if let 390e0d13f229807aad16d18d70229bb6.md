# 강력한 흐름 제어 연산자 match와 if let

- **요약 목록**
    - **`match`**: 열거형의 모든 가능한 가지(`arm`)를 빠짐없이 검사해야 하는 패턴 매칭 연산자입니다. (전수 검사 필수)
    - **값 추출**: 패턴 매칭을 통해 `Option<T>`이나 `Enum` 내부에 바인딩된 실제 데이터를 안전하게 분리해 낼 수 있습니다.
    - **`if let`**: 오직 하나의 특정 패턴에만 관심이 있고, 나머지 경우는 무시하고 싶을 때 사용하는 간결한 문법 설탕(`Syntactic sugar`)입니다.

## 강력한 흐름 제어 연산자 `match`

`match`는 표현식의 값을 일련의 패턴과 비교하여, 일치하는 패턴에 따라 코드를 실행하는 매우 강력한 흐름 제어 연산자입니다. 타 언어의 `switch`와 유사하지만 훨씬 강력하며 안전합니다.

```rust
enum Coin {
    Penny,
    Nickel,
    Dime,
    Quarter,
}

fn value_in_cents(coin: Coin) -> u8 {
    match coin {
        Coin::Penny => 1,
        Coin::Nickel => 5,
        Coin::Dime => 10,
        Coin::Quarter => 25,
    }
}
```

### 1. 전수 검사 (Exhaustiveness)

Rust의 `match`는 **모든 가능한 경우의 수를 반드시 처리**해야 합니다. 만약 `Coin::Quarter` 분기를 빼먹는다면, 컴파일러는 "처리되지 않은 패턴이 있다"며 컴파일 에러를 일으킵니다. 버그를 예방하는 강력한 안전장치입니다.

### 2. 값을 묶어내는 패턴 (Values-binding)

패턴과 일치하는 변형(Variant)이 가지고 있는 내부 데이터를 매칭 과정에서 바로 꺼내어 사용할 수 있습니다. 앞서 배운 `Option<T>`에서 값을 꺼낼 때 가장 많이 사용됩니다.

```rust
fn plus_one(x: Option<i32>) -> Option<i32> {
    match x {
        None => None,
        Some(i) => Some(i + 1), // Some 내부의 값 i를 꺼내어 사용
    }
}
```

### 3. 나머지 모든 매칭을 처리하는 `_` (Placeholder)

모든 값을 일일이 나열하기 힘들 때는 `_` 패턴을 사용하여 "그 외 나머지 전부"를 처리할 수 있습니다.

```rust
let dice_roll = 9;
match dice_roll {
    3 => println!("3이 나왔습니다."),
    7 => println!("7이 나왔습니다."),
    _ => println!("그 외의 숫자가 나왔습니다."), // default 케이스와 동일
}
```

## 간결한 흐름 제어 `if let`

`match` 연산자는 안전하지만, 오직 하나의 조건만 처리하고 싶을 때도 모든 분기(예: `_ => ()`)를 작성해야 하므로 코드가 장황해질 수 있습니다. 이때 사용하는 문법이 `if let`입니다.

```rust
let config_max = Some(3u8);

// 1. match를 사용할 때
match config_max {
    Some(max) => println!("최대치: {max}"),
    _ => (), // 나머지 경우는 아무것도 안 함 (장황함)
}

// 2. if let을 사용할 때 (동일한 동작)
if let Some(max) = config_max {
    println!("최대치: {max}");
}
```

- `if let 패턴 = 표현식` 형태로 사용합니다.
- 지정한 패턴과 일치할 때만 블록이 실행되고, 패턴 내부의 값을 변수(`max`)로 바인딩하여 사용할 수 있습니다.
- 필요한 경우 `else` 블록을 붙여 패턴이 일치하지 않을 때의 예외 처리도 가능합니다.
