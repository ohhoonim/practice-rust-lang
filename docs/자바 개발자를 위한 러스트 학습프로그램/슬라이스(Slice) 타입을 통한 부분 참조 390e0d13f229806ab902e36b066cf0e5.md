# 슬라이스(Slice) 타입을 통한 부분 참조

- **요약 목록**
    - **슬라이스(`Slice`)**: 소유권을 가져오지 않고, 연속된 컬렉션(배열, 문자열 등)의 **일부분만 참조**할 수 있게 해주는 타입입니다.
    - **문자열 슬라이스(`&str`)**: 문자열의 특정 구간을 `[시작_인덱스..끝_인덱스]` 형태로 지정하여 참조하며, 끝 인덱스는 포함되지 않습니다.
    - **안전성 보장**: 슬라이스가 살아있는 동안 원본 컬렉션이 수정되거나 메모리 위치가 바뀌는 것을 대여 규칙을 통해 컴파일 시점에 방지합니다.

## 슬라이스 (Slice)

슬라이스는 전체 컬렉션을 통째로 대여하는 것이 아니라, **연속된 일부분만을 참조**할 수 있는 특별한 참조 타입입니다. 참조이기 때문에 데이터의 소유권을 가지지 않습니다.

가장 흔하게 사용되는 형태가 바로 **문자열 슬라이스**입니다.

## 문자열 슬라이스 (`&str`)

문자열 슬라이스는 `[시작_인덱스..끝_인덱스]` 구문을 사용하여 표현합니다. 이때 `끝_인덱스`는 참조 범위에 **포함되지 않습니다** (이상~미만 방식).

```rust
let s = String::from("hello world");

let hello: &str = &s[0..5];  // "hello" (0, 1, 2, 3, 4 인덱스)
let world: &str = &s[6..11]; // "world" (6, 7, 8, 9, 10 인덱스)
```

### 인덱스 생략 문법

- **0부터 시작**할 때는 앞의 숫자를 생략할 수 있습니다: `&s[..5]`
- **맨 마지막까지 포함**할 때는 뒤의 숫자를 생략할 수 있습니다: `&s[6..]`
- **전체 범위를 슬라이스**할 때는 둘 다 생략할 수 있습니다: `&s[..]`

## 슬라이스의 내부 구조와 안전성

문자열 슬라이스(`&str`)는 내부적으로 **데이터가 시작되는 시작점의 포인터**와 **슬라이스의 길이**만을 가지고 있는 경량 구조체입니다.

이 구조 덕분에 Rust는 런타임 성능 저하 없이 부분 문자열을 안전하게 다룰 수 있으며, 대여 규칙이 그대로 적용됩니다.

```rust
let mut s = String::from("hello world");
let word = &s[0..5]; // 불변 대여 발생!

// s.clear(); // 컴파일 에러! word가 슬라이스로 참조 중이므로 가변 대여로 수정 불가
println!("첫 단어: {word}");
```

만약 원본 문자열을 지워버릴 수 있다면 슬라이스(`word`)는 이미 사라진 메모리를 가리키게 되므로, Rust 컴파일러는 이를 원천 차단합니다.

## 기타 슬라이스 (기본 배열 슬라이스)

문자열뿐만 아니라 일반 배열도 똑같은 방식으로 슬라이스를 만들 수 있습니다. 이때의 타입은 `&[T]` 형태가 됩니다.

```rust
let a = [1, 2, 3, 4, 5];
let slice: &[i32] = &a[1..3]; // [2, 3]을 참조하는 슬라이스
```
