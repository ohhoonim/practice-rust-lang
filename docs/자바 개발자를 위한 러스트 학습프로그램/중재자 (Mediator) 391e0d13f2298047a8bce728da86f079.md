# 중재자 (Mediator)

- **목표:** 여러 객체 간의 복잡한 상호작용을 한 곳으로 집중시켜, 컴포넌트들이 서로 직접 참조하지 않고 느슨한 결합(Loose Coupling)을 유지하도록 하는 중재자 패턴을 구현합니다.
- **조건:** Rust에서는 여러 객체가 서로를 상호 참조하는 다대다($N:N$) 구조를 소유권 규칙상 표현하기 매우 어렵습니다. 따라서 중재자(Mediator) 역할을 하는 구조체가 중심이 되어 메시지 중계 및 제어권을 완전히 통제하는 방식으로 설계를 제한하여 안전하게 구현합니다.

### [연습 문제] 중재자 패턴 구현

채팅방(ChatRoom) 안에서 참가자(User)들이 서로 직접 메시지를 주고받지 않고, 채팅방이라는 중재자를 거쳐 메시지를 송수신하는 시스템을 구현하세요.

1. **중재자(Mediator):** `ChatRoom` 구조체는 참가자들의 이름 목록(`Vec<String>`)을 가집니다.
2. **컴포넌트:** `User` 구조체는 자신의 `name`과 자신이 속한 `ChatRoom`을 가집니다.
3. `User`가 `send_message`를 호출하면, 직접 다른 유저에게 접근하는 것이 아니라 `ChatRoom` 중재자의 `broadcast` 메서드를 호출합니다. 중재자는 발신자를 제외한 모든 참가자에게 메시지가 전송되는 형태의 로그 문자열들을 수집하여 반환합니다.

```rust

struct ChatRoom {
    participants: Vec<String>,
}

impl ChatRoom {
    fn new() -> Self {
        ChatRoom {
            participants: Vec::new(),
        }
    }

// 2. 컴포넌트 객체 (Colleague)
struct User<'a> {
    name: String,
    mediator: &'a ChatRoom,
}

impl<'a> User<'a> {
    fn new(name: &str, mediator: &'a ChatRoom) -> Self {
        User {
            name: name.to_string(),
            mediator,
        }
    }

    // 중재자에게 메시지 방송 처리를 위임합니다.
    fn send_message(&self, message: &str) -> Vec<String> {
        self.mediator.broadcast(&self.name, message)
    }
}

#[cfg(test)]
mod tests {
    use super::*;

    #[test]
    fn test_mediator_pattern() {
        let mut chat_room = ChatRoom::new();
        chat_room.register("Alice");
        chat_room.register("Bob");
        chat_room.register("Charlie");

        let alice = User::new("Alice", &chat_room);

        // Alice가 메시지를 보내면 중재자(ChatRoom)가 Bob과 Charlie에게 중계해야 함
        let logs = alice.send_message("Hello, everyone!");

        assert_eq!(logs.len(), 2);
        assert_eq!(logs[0], "Bob received from Alice: Hello, everyone!");
        assert_eq!(logs[1], "Charlie received from Alice: Hello, everyone!");
    }
}
```
