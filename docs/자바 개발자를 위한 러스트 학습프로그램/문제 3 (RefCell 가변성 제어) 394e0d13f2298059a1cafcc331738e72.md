# 문제 3 (RefCell 가변성 제어)

## [문제 3] RefCell을 활용한 내부 가변성 구현

### 설명

구조체 참조자가 불변(`&self`) 상태임에도 불구하고, 특정 필드의 값을 안전하게 변경해야 할 때 사용하는 `RefCell` 훈련입니다.

메시지 전송 횟수를 기록하는 가짜 로거(Mock Logger)의 카운터를 업데이트하세요.

- `send_log` 메서드는 `&self`를 받으므로 구조체 필드를 직접 수정할 수 없습니다.
- `msg_count` 필드를 `RefCell`로 감싸서 런타임 대여 검사를 이용해 값을 증가시키세요.

### 템플릿 코드

```rust
use std::cell::RefCell;

struct MockLogger {
    msg_count: RefCell<u32>,
}

impl MockLogger {
    fn new() -> Self {
        MockLogger {
            msg_count: RefCell::new(0),
        }
    }

    fn send_log(&self, _msg: &str) {
        // 이 부분을 구현하세요.
        // borrow_mut()를 사용하여 내부 값을 1 증가시킵니다.
    }

    fn get_count(&self) -> u32 {
        *self.msg_count.borrow()
    }
}

#[cfg(test)]
mod tests {
    use super::*;

    #[test]
    fn test_refcell_mutability() {
        let logger = MockLogger::new();

        // 불변 객체인 logger를 사용해 함수를 호출하지만 내부 카운트가 올라가야 함
        logger.send_log("First warning");
        logger.send_log("Second warning");

        assert_eq!(logger.get_count(), 2);
    }
}
```
