# 열거형(Enum)과 데이터 매칭

- **요약 목록**
    - **열거형(`Enum`)**: 어떤 값이 여러 가능한 변형(Variant) 중 하나임을 나타내는 타입입니다. Rust의 열거형은 각 변형에 **다양한 종류의 데이터를 직접 포함**할 수 있습니다.
    - **Option 열거형**: Rust에는 Null이 없습니다. 대신 값이 있거나(`Some(T)`) 없을 수 있음(`None`)을 나타내는 `Option<T>` 열거형을 표준 라이브러리로 제공합니다.

## 열거형 (Enum) 정의 및 데이터 포함

열거형은 가능한 값들을 열거하여 하나의 타입을 정의할 수 있게 해줍니다. Rust의 열거형은 타 언어와 달리, **각 변형(Variant)에 서로 다른 타입과 크기의 데이터를 바인딩**할 수 있는 매우 강력한 기능을 가지고 있습니다.

### 1. 다양한 데이터를 가지는 열거형 정의

```rust
enum Message {
    Quit,                       // 데이터가 없음
    Move { x: i32, y: i32 },    // 익명 구조체를 포함
    Write(String),              // String 하나를 포함
    ChangeColor(i32, i32, i32), // 세 개의 i32 튜플을 포함
}
```

### 2. 열거형 값 생성

열거형의 각 변형은 식별자 뒤에 콜론 두 개(`::`)를 붙여서 생성합니다. 전혀 다른 형태의 데이터들이지만 모두 `Message`라는 **동일한 타입**으로 취급됩니다.

```rust
let m1 = Message::Quit;
let m2 = Message::Move { x: 10, y: 20 };
let m3 = Message::Write(String::from("hello"));
```

## Rust의 혁신: `Option` 열거형

Rust에는 수많은 소프트웨어에서 버그를 유발하는 `Null`(값이 없음) 개념이 존재하지 않습니다. 대신 표준 라이브러리에서 값이 구현되어 있거나 없을 수 있는 문맥을 나타내는 `Option<T>` 열거형을 제공합니다.

```rust
// 표준 라이브러리에 내장된 Option 열거형의 구조
enum Option<T> {
    None,
    Some(T),
}
```

- `None`: 값이 없는 상태 (타 언어의 Null에 대응)
- `Some(T)`: `T` 타입의 어떤 값이 존재하는 상태

```rust
let some_number = Some(5);
let some_char = Some('e');
let absent_number: Option<i32> = None; // None일 때는 타입을 명시해야 함
```

### `Option`을 사용하는 이유

Rust에서는 일반 타입(예: `i32`)에 절대 Null이 들어올 수 없습니다. 값이 없을지도 모르는 상황이라면 무조건 `Option<i32>` 타입으로 강제됩니다.

따라서 개발자는 `Some` 안에 있는 실제 값을 꺼내 쓰기 전에, **값이 없을 때(`None`)에 대한 처리를 컴파일 시점에 반드시 강제**당하게 되므로 메모리 안전성이 극대화됩니다. 이 값을 안전하게 꺼내고 매칭하는 방법이 바로 다음 순서인 `match` 연산자입니다.
