# 상수(const)와 섀도잉(Shadowing)

- **요약 목록**
    - **상수(`const`)**: 프로그램 전체 생명주기 동안 절대 변경되지 않는 값으로, 컴파일 타임에 결정되며 반드시 타입을 명시해야 합니다.
    - **섀도잉(`Shadowing`)**: 기존 변수와 동일한 이름으로 새 변수를 선언하여 이전 변수를 가리는 기법으로, 타입 변경이 가능하며 변수의 불변성을 유지할 수 있습니다.

## 상수 (Constants)

상수는 가변성 자체가 허용되지 않는 값입니다. `mut`을 사용할 수 없으며, 프로그램이 실행되는 동안 절대 바뀌지 않습니다.

```rust
const THREE_HOURS_IN_SECONDS: u32 = 3 * 60 * 60;
```

- `let` 대신 `const` 키워드를 사용합니다.
- **타입 선언이 필수**입니다. 컴파일러가 타입을 추론하지 않습니다.
- 상수는 고정된 값뿐만 아니라 컴파일 타임에 계산될 수 있는 상수 표현식도 할당할 수 있습니다.

## 섀도잉 (Shadowing)

Rust에서는 먼저 선언한 변수와 **동일한 이름의 새 변수를 다시 선언**할 수 있습니다. 이를 '이전 변수를 가린다'고 하여 섀도잉이라고 부릅니다.

```rust
let x = 5;
let x = x + 1;
```

### 섀도잉의 특징

1. **`mut`과의 차이점**: `mut` 없이도 값을 새로 할당하는 것처럼 보이지만, 실제로는 기존 변수를 가리고 새 변수를 만드는 것입니다. `let`을 떼고 값을 재할당하면 컴파일 에러가 나지만, `let`을 다시 사용하면 합법적으로 새 값을 바인딩할 수 있습니다.
2. **타입 변환 가능**: 변수의 이름은 유지하면서 전혀 다른 데이터 타입의 값을 저장할 수 있습니다.

```rust
let spaces = "   "; // 문자열 슬라이스 타입
let spaces = spaces.len(); // 정수 타입으로 재선언 가능
```
