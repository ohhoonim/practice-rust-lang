# 문제 2 (Result 에러 전파)

## [문제 2] Result 타입과 ? 연산자를 이용한 에러 전파

### 설명

자바의 `throws Exception` 처럼, 함수 실행 중 실패할 수 있는 상황을 `Result` 타입으로 명시하고 `?` 연산자로 에러를 상위 함수로 던지는 훈련입니다.

입력받은 문자열 숫자를 파싱하여 제곱한 값을 반환하는 함수를 작성하세요.

- 파싱에 성공하면 제곱한 값을 `Ok`에 담아 반환합니다.
- 파싱에 실패하면 발생한 에러(`ParseIntError`)를 `?` 연산자로 즉시 상위로 반환합니다.

### 템플릿 코드

```rust
use std::num::ParseIntError;

fn square_number(num_str: &str) -> Result<i32, ParseIntError> {
    // 이 부분을 구현하세요.
    // 문자열을 정수로 파싱(.parse::<i32>())할 때 발생하는 에러를 ? 연산자로 처리하세요.
}

#[cfg(test)]
mod tests {
    use super::*;

    #[test]
    fn test_result_propagation() {
        // 성공 케이스: "12" -> 12 -> 제곱해서 144
        assert_eq!(square_number("12"), Ok(144));

        // 실패 케이스: 정수로 바꿀 수 없는 문자열인 경우 에러 반환 검증
        assert!(square_number("abc").is_err());
    }
}
```
