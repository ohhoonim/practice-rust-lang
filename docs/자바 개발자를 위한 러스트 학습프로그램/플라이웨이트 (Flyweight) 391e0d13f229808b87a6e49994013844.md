# 플라이웨이트 (Flyweight)

- **목표:** 수많은 유사 객체들이 가질 수 있는 불변의 공통 데이터(Intrinsic State)를 싱글톤 패턴 형태의 팩토리를 통해 공유(Sharing)함으로써, 가변 데이터(Extrinsic State)만 개별적으로 다루어 메모리 사용량을 최소화하는 플라이웨이트 패턴을 구현합니다.
- **조건:** Rust에서는 고유 상태를 보관하는 플라이웨이트 객체를 공유하기 위해 `std::collections::HashMap`과 참조 카운팅 스마트 포인터인 `std::rc::Rc`를 조합하여 공유 참조 인스턴스를 안전하게 관리하는 패턴을 구현합니다.

### [연습 문제] 플라이웨이트 패턴 구현

메모리를 많이 차지하는 나뭇잎 이미지 데이터(고유 상태)를 공유하고, 나무의 위치 정보(외래 상태)는 개별적으로 관리하는 산림(Forest) 시뮬레이션 시스템을 구현하세요.

1. **Flyweight:** `TreeType` 구조체는 나무의 공유 가능한 속성인 이름(`name`)과 무거운 텍스처 데이터(`texture`)를 가집니다.
2. **Flyweight Factory:** `TreeFactory` 구조체는 `HashMap<String, Rc<TreeType>>`을 내부에 두고, 동일한 이름의 나무 타입 요청이 오면 기존 객체의 `Rc`를 반환하고 없으면 새로 생성하여 캐싱합니다.
3. **Context:** `Tree` 구조체는 개별 나무의 위치 좌표(`x`, `y`)와 공유된 `Rc<TreeType>`을 가집니다.

```rust
use std::collections::HashMap;
use std::rc::Rc;

// 1. 공유될 고유 상태 (Intrinsic State)
#[derive(Debug, PartialEq)]
struct TreeType {
    name: String,
    texture: Vec<u8>, // 무거운 데이터라 가정
}

// 2. 플라이웨이트 팩토리 (Flyweight Factory)
struct TreeFactory {
    tree_types: HashMap<String, Rc<TreeType>>,
}

impl TreeFactory {
    fn new() -> Self {
        TreeFactory {
            tree_types: HashMap::new(),
        }
    }

    // get_tree_type 메서드를 구현하세요.
    // tree_types 해시맵에 해당 name이 이미 존재하면 그 Rc<TreeType>을 복사해서 반환합니다.
    // 존재하지 않는다면, texture에 값으로 [1, 2, 3, 4]를 채운 새로운 TreeType을 만들어 해시맵에 삽입 후 반환합니다.
    fn get_tree_type(&mut self, name: &str) -> Rc<TreeType> {
        // 이 부분을 구현하세요.
    }
}

// 3. 외래 상태를 포함하는 컨텍스트 객체 (Context)
struct Tree {
    x: i32,
    y: i32,
    tree_type: Rc<TreeType>,
}

impl Tree {
    fn new(x: i32, y: i32, tree_type: Rc<TreeType>) -> Self {
        Tree { x, y, tree_type }
    }
}

#[cfg(test)]
mod tests {
    use super::*;

    #[test]
    fn test_flyweight_pattern() {
        let mut factory = TreeFactory::new();

        // 동일한 종류의 나무 타입을 팩토리로부터 획득
        let type_oak1 = factory.get_tree_type("Oak");
        let type_oak2 = factory.get_tree_type("Oak");
        let type_pine = factory.get_tree_type("Pine");

        // 두 Oak 타입은 물리적으로 동일한 메모리 주소를 가리키고 있는지 검증 (공유 확인)
        assert!(Rc::ptr_eq(&type_oak1, &type_oak2));
        assert!(!Rc::ptr_eq(&type_oak1, &type_pine));

        // 외래 상태(좌표)를 부여하여 개별 객체 생성
        let forest = vec![
            Tree::new(10, 20, type_oak1),
            Tree::new(50, 60, type_oak2),
            Tree::new(100, 200, type_pine),
        ];

        assert_eq!(forest[0].tree_type.name, "Oak");
        assert_eq!(forest[1].tree_type.name, "Oak");
        assert_eq!(forest[2].tree_type.name, "Pine");

        // 팩토리 내부에 캐싱된 TreeType은 총 2개여야 함
        assert_eq!(factory.tree_types.len(), 2);
    }
}
```
