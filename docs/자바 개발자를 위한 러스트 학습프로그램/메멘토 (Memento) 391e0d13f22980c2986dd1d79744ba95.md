# 메멘토 (Memento)

- **목표:** 객체의 내부 상태를 캡슐화 규칙을 위반하지 않고 외부(Memento)에 저장하여, 나중에 원하는 상태로 복구(Rollback)할 수 있는 메멘토 패턴을 구현합니다.
- **조건:** Rust에서는 메멘토 객체의 내부 데이터를 프라이빗하게 유지하면서 원본 객체(Originator)만 접근할 수 있도록 가시성(Visibility)과 모듈 제어를 활용하여 구현합니다.

### [연습 문제] 메멘토 패턴 구현

텍스트 편집기의 상태를 저장하고 이전 상태로 되돌리는 스냅샷 기능을 메멘토 패턴으로 구현하세요.

1. **Originator (원조자):** `Editor` 구조체는 현재 텍스트(`content`) 상태를 가지며, 메멘토를 생성하거나 메멘토로부터 상태를 복구합니다.
2. **Memento (메멘토):** `EditorMemento` 구조체는 저장된 텍스트 상태를 보관하는 불변 객체입니다.
3. `Editor`에 `save` 메서드를 구현하여 현재 상태를 담은 `EditorMemento`를 반환하게 하고, `restore` 메서드를 구현하여 메멘토를 전달받아 상태를 이전으로 되돌리도록 만드세요.

#[cfg(test)]
mod tests {
    use super::*;

    #[test]
    fn test_memento_pattern() {
        let mut editor = Editor::new();

        // 1. 첫 번째 상태 작성 후 저장
        editor.type_text("Hello, ");
        let saved_state1 = editor.save();

        // 2. 두 번째 상태 작성 후 저장
        editor.type_text("World!");
        let saved_state2 = editor.save();

        // 3. 추가 작성
        editor.type_text(" Extra text.");
        assert_eq!(editor.content, "Hello, World! Extra text.");

        // 4. 두 번째 저장 시점으로 복구 (Hello, World!)
        editor.restore(&saved_state2);
        assert_eq!(editor.content, "Hello, World!");

        // 5. 첫 번째 저장 시점으로 복구 (Hello, )
        editor.restore(&saved_state1);
        assert_eq!(editor.content, "Hello, ");
    }
}
