# 소유권의 세 가지 원칙과 메모리 관리 방식

- **요약 목록**
    - **메모리 관리 방식**: 가비지 컬렉터(GC)나 수동 해제(malloc/free) 없이, **소유권 시스템**을 통해 컴파일 시점에 메모리 해제 타이밍을 결정합니다.
    - **세 가지 원칙**:
        1. Rust의 모든 값은 소유자(Owner)라 불리는 변수가 있습니다.
        2. 소유자는 오직 **단 하나**만 존재할 수 있습니다.
        3. 소유자가 스코프(Scope)를 벗어나면, 그 값은 자동으로 **메모리에서 해제**됩니다.

## Rust의 메모리 관리 방식

기존 프로그래밍 언어들은 메모리를 관리하기 위해 두 가지 방식 중 하나를 주로 사용했습니다.

1. **가비지 컬렉터 (Garbage Collector, GC)**: 자바나 고(Go)처럼 프로그램 실행 중에 사용되지 않는 메모리를 자동으로 찾아내어 해제하는 방식입니다. 편리하지만 런타임 성능 저하(Stop-the-world 등)가 발생할 수 있습니다.
2. **명시적 메모리 관리**: C/C++처럼 개발자가 직접 메모리를 할당(`malloc`)하고 해제(`free`)하는 방식입니다. 성능은 극대화되지만, 해제를 까먹거나(Memory Leak) 이미 해제된 메모리에 접근하는(Dangling Pointer) 치명적인 버그가 발생하기 쉽습니다.

Rust는 제3의 길인 **소유권(Ownership) 시스템**을 사용합니다. 컴파일러가 코드를 빌드할 때 소유권 규칙을 철저히 검사하여, **프로그램이 실행되는 동안 성능 저하 없이, 안전하게 메모리가 자동으로 해제**되도록 만듭니다.

## 소유권의 세 가지 원칙

Rust 코드 전체를 지배하는 소유권의 핵심 규칙 세 가지는 다음과 같습니다.

### 원칙 1: Rust의 각 값은 소유자(Owner)가 있습니다.

여기서 소유자는 값이 할당된 **변수**를 의미합니다.

### 원칙 2: 값의 소유자는 한 번에 딱 하나만 존재할 수 있습니다.

여러 변수가 하나의 메모리 공간을 동시에 완전히 소유할 수 없습니다. 이 규칙 덕분에 데이터 경합이나 메모리 오염이 원천 차단됩니다. (값의 복사나 대여는 가능하지만, 완전한 소유는 하나만 가능합니다.)

### 원칙 3: 소유자가 스코프(Scope)를 벗어나면, 값은 제거됩니다.

변수가 선언된 중괄호 `{}` 블록이 끝나는 순간, 변수가 소유하고 있던 메모리 공간은 Rust에 의해 자동으로 안전하게 반환됩니다. 이를 전문 용어로 **`drop` 함수가 호출된다**고 표현합니다.

```rust
{                      // s는 아직 선언되지 않아 유효하지 않음
    let s = "hello";   // s는 이 지점부터 유효함
    // s를 가지고 무언가 수행
}                      // 이 스코프가 종료되면 s는 더 이상 유효하지 않으며 메모리에서 해제됨(drop)
```
