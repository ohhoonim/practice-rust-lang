# 이동(Move)과 복사(Copy)의 개념

- **요약 목록**
    - **이동(`Move`)**: 힙(Heap) 메모리를 사용하는 타입(예: `String`)의 소유권을 다른 변수로 넘기는 행위입니다. 이동 후 기존 변수는 사용할 수 없습니다.
    - **복사(`Copy`)**: 스택(Stack) 메모리만 사용하는 고정 크기 타입(예: 정수, 부울)은 기존 값을 그대로 복제하여 새 변수를 만듭니다. 기존 변수도 계속 사용할 수 있습니다.

## 1. 이동 (Move)

힙(Heap) 메모리를 사용하여 크기가 유동적인 데이터 구조는 값을 다른 변수에 대입할 때 **소유권이 이동**합니다.

```rust
let s1 = String::from("hello");
let s2 = s1; // s1의 데이터 소유권이 s2로 이동(Move)함

// println!("{s1}"); // 컴파일 에러! s1은 더 이상 유효하지 않음
```

Rust는 성능을 위해 힙 메모리에 있는 실제 문자열 데이터를 통째로 복사하지 않고, 데이터의 주소·길이·용량이 담긴 메타데이터만 `s2`로 복사합니다.

이때 만약 `s1`과 `s2`가 같은 메모리를 가리키고 있는 상태에서 스코프가 끝나면, 두 변수가 같은 메모리를 두 번 해제하는 **Double Free** 오류가 발생할 수 있습니다. Rust는 이를 방지하기 위해 소유권을 `s2`로 넘기는 순간 **`s1`을 무효화(Move)** 시켜버립니다.

## 2. 복사 (Copy)

반면, 컴파일 시점에 크기가 고정되어 스택(Stack) 메모리에 완전히 저장되는 기본 타입들은 값이 다른 변수에 대입될 때 **실제 값이 복사**됩니다.

```rust
let x = 5;
let y = x; // x의 값이 y로 복사(Copy)됨

println!("x = {x}, y = {y}"); // 정상 작동! x와 y 모두 사용 가능
```

이 타입들은 `Copy` 트레이트(Trait)라는 특별한 주석이 구현되어 있습니다. 스택 메모리에서의 복사는 매우 빠르고 안전하기 때문에, 소유권을 이동시켜 기존 변수를 무효화할 필요가 없습니다.

### `Copy`가 가능한 대표적인 타입들

- 모든 정수형(예: `u32`, `i32`) 및 부동 소수점형(`f64`)
- 부울형(`bool`)
- 문자형(`char`)
- `Copy` 가능한 타입들로만 이루어진 튜플 (예: `(i32, i32)`는 가능, `(i32, String)`은 불가능)
